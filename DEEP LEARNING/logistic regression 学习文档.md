# Logistic Regression 学习文档

这份代码是按照吴恩达深度学习课程里讲逻辑回归的主线整理的：

`输入 X -> 线性部分 Z = w^T X + b -> sigmoid -> 得到预测概率 A -> 计算损失 -> 梯度下降更新参数`

---

## 1. 逻辑回归在做什么

逻辑回归本质上是在做二分类。

比如：

- 图片是不是猫
- 邮件是不是垃圾邮件
- 一个样本是不是正类

它先算一个线性结果：

```text
z = w^T x + b
```

再经过 `sigmoid` 函数，把结果压缩到 `0 ~ 1` 之间：

```text
a = sigmoid(z) = 1 / (1 + e^(-z))
```

这里的 `a` 可以理解为：

`P(y=1 | x)`

也就是“这个样本属于 1 类的概率”。

## 2. 代码结构对应课程思路

你现在的 Python 文件是：

[logistic regression.py](D:/py/DEEP%20LEARNING/logistic%20regression.py)

里面我按下面的顺序写好了：

### `sigmoid(z)`

把线性输出变成概率。

### `initialize_parameters(dim)`

初始化参数：

- `w = 0`
- `b = 0`

这和吴恩达课程里一开始的写法一致。

### `propagate(w, b, X, Y)`

这个函数最关键，里面同时做了两件事：

1. 前向传播
2. 反向传播

前向传播：

```text
Z = w^T X + b
A = sigmoid(Z)
cost = 交叉熵损失
```

反向传播：

```text
dw = 1/m * X * (A - Y)^T
db = 1/m * sum(A - Y)
```

这里的 `dw` 和 `db` 就是梯度下降要用的导数。

### `optimize(...)`

循环很多次：

- 先调用 `propagate`
- 算出当前损失和梯度
- 再更新 `w` 和 `b`

更新公式：

```text
w = w - learning_rate * dw
b = b - learning_rate * db
```

### `predict(w, b, X)`

先得到预测概率，再用阈值 `0.5` 转成分类结果：

- 概率 >= 0.5，预测为 `1`
- 概率 < 0.5，预测为 `0`

### `model(...)`

把前面的步骤串起来，形成一个完整训练流程。

## 3. 为什么 X 的形状是 `(特征数, 样本数)`

吴恩达课程里经常使用这种矩阵形式：

- `X.shape = (n_x, m)`
- `Y.shape = (1, m)`

其中：

- `n_x` 是特征数
- `m` 是样本数

比如二维特征、8 个样本时：

```text
X.shape = (2, 8)
Y.shape = (1, 8)
```

这样写的好处是可以直接做向量化运算，不需要一个样本一个样本地写循环。

## 4. 这份代码里你最应该重点看什么

建议你先重点理解这 4 行：

```python
Z = np.dot(w.T, X) + b
A = sigmoid(Z)
dw = 1 / m * np.dot(X, (A - Y).T)
db = 1 / m * np.sum(A - Y)
```

因为这 4 行基本就是逻辑回归最核心的数学实现。

你可以这样记：

- `Z`：线性计算结果
- `A`：预测概率
- `dw`：损失函数对 `w` 的导数
- `db`：损失函数对 `b` 的导数

## 5. 你可以怎么练习

建议你按下面顺序练：

1. 先自己手推一个样本的 `z` 和 `a`
2. 再看代码里矩阵版的 `Z` 和 `A`

4. 修改 `learning_rate`，观察损失下降快慢
5. 把 `print_cost=True` 打开，看损失是否在不断下降
